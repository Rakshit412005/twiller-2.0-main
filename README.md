Twiller Website Tasks Flow 

TASK 1 - 

MOST IMPORTANT- The user have to enable browser notification on his system from settings in Laptop/computer otherwise task 1 will not work.

After the user logs in, the first page on the screen will be Feed page where "Enable Notification" is clearly visible in front above tweet creating box, After clicking the "Enable Notification" button a browser pop up will appear to ask permission for enabling notification choose "While visiting this site" and then a toast success message will appear saying "Notifications Enabled" and then if the user created a post which has keywords "cricket" and "science" then  the browser's built-in Notification will appear showing the full tweet. The user can disable the notifications by clicking the same which is no saying "Disabled Notifications" and after clicking "Disabled Notifications" button browser built in notifications will be disabled and no notifications will be showed even if the user creates a tweet with keywords "cricket" and "science".

TASK 2 - 

On the Feed page itself there is one block where the user can create tweets there only one MIC icon is present for uploading and recording audios, When the user clicks that MIC icon a menu appears which has two buttons  "Request OTP" and another "Verify OTP" between them there is one input box to enter OTP, the OTP will be sent on user's gmail and after verifying the OTP the user can record and upload audio. Audio can be uploaded or recorded only between 2 PM to 7PM outside this time period user will not be allowed to upload or record audio. Also the audio should be less than 5 minutes and 100mbs exceeding this limit will result in Toast error message. 

TASK 3 - 

NOTE - In order to verify Task 3 create a separate new account on Twiller website using Email and Password , do not use that email which is already been used for "Sign up with google" method

On the very first page of the website which the user sees is the Login page there only under "Create account" button "Forgot password" button is present, when the user clicks on that button it gets routed to forgot-password page where a menu box will appear where the user can enter the email whose password he wants to change , after clicking "Send Reset Email" a mail will be sent to the user's mail which he has entered, The mail will surely be present in the SPAM section of the Gmail with name "no reply" please mark it as "NOT SPAM" for future convenience. The mail contains the link of the reset-password page of the Twiller website on clicking on that will open the reset-password page where the user can reset password, Password generator functionality is also provided to generate new strong passwords. Once the user enters a new password  a toast success message will appear saying "Password changed successfully" 
NOTE - After that the user can close that reset-password tab and can reroute one tab backwards from forgot-password page to login page and continue with new password.

TASK 4 - 

On the right sidebar there is one section displaying "Subscription" with a Subscribe" button , when the user clicks on that "Subscribe" button he will be routed to page which contains 3 PLANS - Bronze , Silver and Gold , when the user choose any one plan RazorPay's test mode will be triggered, select any payment method and choose "Success" for making payment, after that payment will be successful and user will rerouted to that PLANS page. Also a Invoice mail will be sent to user's mail. The user can buy subscription only between 10AM to 11AM outside that time period payments will not be allowed.

NOTE - From here the user can go just one tab backwards and he can see its plan got updated displaying on the right sidebar.

TASK 5 - 

After the user logs in on the left side bar he can clearly see 6 language options, on clicking French an OTP will be sent to user's gmail and after verifying OTP Twiller website's UI will be changed into French, for other languages the user has to enter a mobile number and otp will be sent on that number after OTP verification Twiller website UI will be changed to the required chosen language.


TASK 6 - 

When the user tries logs in on chrome, a OTP will be sent to user's gamil and verify OTP page will appear , without OTP verification the user cannot access the website, after OTP verification the user's login history will be stored inside "Security" tab.
Now inorder to check the "Security" tab on the left sidebar click on  "Profile" which is present between "Bookmarks" and "More" after that user can see 6 tabs - Posts, Replies, Highlights, Articles, Media and Security click on "Security" to see user's login information. other browsers like Edge, Firefox etc will not require OTP verification. Mobile users will only be allowed to access/login website between 10 AM to 1 PM.

NOTE - Sometimes if Security tab is not updated, the user should just refresh page ( CTRL + R ) and then the security tab will be updated.
